Python工程的组织结构包、模块、类。

包：可以理解为“文件夹”。模块：就是一个一个的文件（.py）。类。函数、变量。

一个包可以包含很多模块，一个模块可以包含很多类。

包的名字就是文件夹的名字；模块的名字就是文件的名字。

seven.c4
six.c4
# 命名空间

如果想让一个普通的文件夹变成python的包，必须在这个文件夹下包含一个 __init__.py 文件。这个文件本身也是一个模块，它的名字比较特殊，就是包名。

import temp.c3 as m

print(m.b)

__init__.py 的用法

当一个包被导入时，python自动运行__init__.py文件。

'''
	__init__.py 文件
'''
import sys
import io
import datetime

__all__ = ['c3', 'c6'] # 允许访问的模块列表list

包和模块是不会重复导入的。注意避免循环导入。

模块内置变量

'''
	this is a doc
'''
print('name: ' + __name__)
print('package: ' + (__package__ or '不属于任何包'))
print('doc: ' + __doc__)
print('file: ' + __file__)

# name: __main__ #单一入口文件就是 __main__
# package: 不属于任何包
# doc:
#         this is a doc

# file: E:\gitProject\python\seven\c8.py

入口文件和普通模块内置变量的区别

入口文件的__name__是__main__，而不是他当前模块的名字。入口文件的package是NoneType，当前模块不属于任何包。

__name__的经典应用

if __name__ == '__main__':
	print('this is a app')

print('this is a module')

相对导入和绝对导入

top-level package顶级包的概念，与入口文件相关。入口文件最好使用绝对导入。

绝对导入：一定要从顶级包开始。
